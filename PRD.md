# PRD: NoteLabs (노트랩) 웹앱 (MVP)

## 1. 문서 개요
- 문서 목적: IDEATION을 기반으로 MVP 제품 개발에 필요한 요구사항을 명확히 정의한다.
- 대상 독자: PM, 디자이너, 프론트엔드/백엔드 엔지니어, QA.
- 기준 버전: v1.0 (2026-07-11)
- 배포/운영 플랫폼: Microsoft Azure

## 2. 제품 비전
학생과 직장인이 태블릿/노트북 환경에서 개인 필기를 빠르고 안정적으로 수행하고, 계정 기반 동기화와 PDF 내보내기를 통해 학습/업무 흐름을 이어갈 수 있는 NoteLabs(노트랩) 웹앱을 제공한다.

## 3. 목표와 성공 지표
### 3.1 제품 목표
1. 자유 필기 중심의 개인 노트 작성 경험 제공
2. 멀티 디바이스 동기화 기반의 연속 사용성 확보
3. PDF 내보내기 제공으로 실사용 가치 확보

### 3.2 KPI/SLO (MVP)
- 활성 지표
  - 주간 활성 사용자(WAU)
  - 사용자당 주간 노트 생성 수
  - PDF 내보내기 사용률
- 성능 SLO
  - 펜 입력 렌더링 지연 p95 <= 30ms
  - 자동 저장 반영 시간 p95 <= 2초
  - 동기화 반영 시간 p95 <= 5초
  - 노트 대시보드 초기 로드 p95 <= 2.5초 (4G/중급 기기 기준)
- 안정성 SLO
  - 자동 저장 실패율 < 0.5%
  - 동기화 충돌 시 데이터 유실 0건 (충돌 사본 생성 정책)

### 3.3 KPI 목표값 (MVP)
| KPI | Baseline | Target | 측정 방식 | 오너 |
|---|---:|---:|---|---|
| 기기 전환 후 10초 내 작업 재개율 | TBD | 95% | 디바이스 전환 후 첫 편집 이벤트까지 시간 | Frontend |
| 충돌 해결 완료율 | TBD | 99% | 충돌 화면 진입 대비 병합 완료 비율 | Backend |

| 크래시 프리 세션 | TBD | 99.5% | 전체 세션 대비 비정상 종료 없는 세션 비율 | DevOps |

## 4. 타깃 사용자
- 1차 타깃: 학생, 직장인
- 핵심 사용 디바이스
  - 태블릿: Galaxy Tab + S Pen 중심
  - 노트북: 마우스/터치패드 중심
- 스타일러스 전략
  - 중요하지만 2순위
  - MVP는 실사용 충분 수준 필기감 확보가 목표

## 5. 제품 범위
### 5.1 In Scope (MVP)
1. 이메일 기반 계정
2. 노트 CRUD (생성/삭제/이름변경)
3. 에디터 캔버스
  - 펜, 형광펜, 지우개
  - 되돌리기/다시하기
  - 올가미 선택(선택/이동/복제)
  - 도형 자동보정(원/직선/사각형)
  - 손글씨 안정화 옵션
4. 자동 저장
5. 멀티 디바이스 동기화
6. PDF 내보내기
7. 한국어/영어 UI
8. Chrome/Safari 핵심 플로우 지원

### 5.2 Out of Scope (MVP)
1. 실시간 협업
2. 공유 링크
3. PNG/JPG 내보내기
4. 유료 플랜/결제
5. 고급 OCR/텍스트 변환
6. 오디오 녹음 연동

### 5.3 확장 범위 (Post-MVP 후보)
1. PDF 열기/수정 (주석 중심)
2. iPadOS Safari 최적화 고도화
3. 고급 손글씨 인식 및 검색

## 6. 핵심 사용자 시나리오
1. 사용자가 로그인한다.
2. 대시보드에서 노트를 생성한다.
3. 에디터에서 필기/편집 도구를 사용해 내용을 작성한다.
4. 자동 저장으로 내용이 보존된다.
5. 다른 기기에서 동일 계정으로 접속해 이어서 편집한다.
6. 노트를 PDF로 내보낸다.

## 7. 기능 요구사항

### FR-01 계정 및 인증
- 설명: 사용자는 이메일/비밀번호로 가입 및 로그인할 수 있어야 한다.
- 상세 요구사항
  1. 회원가입, 로그인, 로그아웃 제공
  2. 인증 실패 시 사용자 친화적 오류 메시지 제공
  3. 사용자별 데이터 접근 제한
- 수용 기준
  - 정상 로그인 시 대시보드 진입
  - 비로그인 상태에서 에디터 접근 불가

### FR-02 노트 관리
- 설명: 사용자는 노트를 생성/삭제/이름변경할 수 있어야 한다.
- 상세 요구사항
  1. 대시보드에서 노트 목록 표시
  2. 새 노트 생성 즉시 에디터 진입 가능
  3. 노트 삭제 시 확인 절차 제공
- 수용 기준
  - CRUD 결과가 즉시 UI에 반영

### FR-03 필기 도구
- 설명: 기본 필기 작성 도구를 제공해야 한다.
- 상세 요구사항
  1. 펜/형광펜/지우개 제공
  2. 색상/두께 프리셋 제공
  3. 줌/팬 상태에서도 필기 정확도 유지
- 수용 기준
  - 도구 전환 후 즉시 입력 가능

### FR-04 편집 기능
- 설명: 사용자 편집 복구 및 선택 조작이 가능해야 한다.
- 상세 요구사항
  1. Undo/Redo 지원
  2. 올가미 선택 후 이동/복제 지원
  3. 도형 자동보정(원/직선/사각형)
- 수용 기준
  - 직전 수정 이력 정확 복원
  - 올가미 선택 객체 이동/복제 정상 동작

### FR-05 필기 품질 개선
- 설명: 손떨림 완화를 위한 선 안정화 옵션 제공.
- 상세 요구사항
  1. 안정화 on/off 옵션
  2. 설정 변경 즉시 이후 입력에 적용
- 수용 기준
  - 안정화 활성 시 평균 선 흔들림 감소(내부 기준 측정)

### FR-06 저장
- 설명: 편집 내용이 자동으로 저장되어야 한다.
- 상세 요구사항
  1. 편집 후 짧은 주기로 자동 저장
  2. 저장 상태 표시(저장 중/저장됨/실패)
  3. 재접속 시 최근 저장 상태 복원
- 수용 기준
  - 브라우저 새로고침 후 직전 저장 상태 확인 가능

### FR-07 동기화
- 설명: 동일 계정 간 노트 동기화를 제공해야 한다.
- 상세 요구사항
  1. 태블릿/노트북 간 최신 데이터 동기화
  2. 충돌 발생 시 충돌 사본 생성
- 수용 기준
  - 충돌 시 원본 유실 없이 사본 확인 가능

### FR-08 내보내기
- 설명: 노트를 PDF로 내보낼 수 있어야 한다.
- 상세 요구사항
  1. 단일 노트 PDF 다운로드
  2. 내보낸 PDF에서 필기 내용이 시각적으로 유지
- 수용 기준
  - 에디터에서 3클릭 이내 내보내기 완료

### FR-09 다국어
- 설명: 한국어/영어 UI를 지원해야 한다.
- 상세 요구사항
  1. 설정에서 언어 전환 가능
  2. 주요 화면 문자열 100% 번역
- 수용 기준
  - 언어 전환 시 즉시 UI 반영

### FR-10 브라우저 호환
- 설명: Chrome/Safari에서 핵심 기능이 동작해야 한다.
- 상세 요구사항
  1. 로그인, 필기, 저장, 동기화, PDF 내보내기 동작
  2. 포인터 이벤트/캔버스 이슈 대응
- 수용 기준
  - 정의된 회귀 테스트 통과

## 8. 비기능 요구사항
### 8.1 성능
1. 펜 렌더링 지연 p95 <= 30ms
2. 자동 저장 p95 <= 2초
3. 동기화 반영 p95 <= 5초

### 8.5 OCR 품질 기준 (MVP)
1. 한글 문자 정확도 >= 94%
2. 영문 문자 정확도 >= 96%
3. 숫자 인식 정확도 >= 98%
4. 혼합 수식/필기 검색 성공률 >= 85%
5. 측정 방식: 언어별 100샘플 수동 검수(월 1회)

### 8.2 안정성
1. 저장 실패 시 재시도 및 사용자 고지
2. 앱 재진입 시 마지막 편집 상태 복원

### 8.3 보안
1. HTTPS 강제
2. 비밀번호 해시 저장
3. 사용자별 접근 제어
4. 비밀/키는 Azure Key Vault로 관리
5. Azure 호스팅 워크로드 간 인증은 Managed Identity 우선

### 8.4 접근성/사용성
1. 핵심 액션 버튼 명확한 라벨 제공
2. 입력 장치(마우스/터치/펜)별 기본 사용 가능

## 16. 릴리스 기준 (Go/No-Go)
### Go 조건
1. Must 기능 100% 동작
2. 성능 SLO 90% 이상 충족
3. 치명적 버그(데이터 유실/로그인 불가) 0건
4. Chrome/Safari 핵심 시나리오 통과
